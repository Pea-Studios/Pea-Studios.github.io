About Pea Studios

Pea Studios started in 2026 because every land claiming plugin was either abandoned, overpriced, or needed 20 config files to set up. So I built PeaTowny as an alternative to Towny and Lands, and it kind of grew from there into a full plugin studio.

All the plugins use commands you already know from Towny and Lands, so theres basically no learning curve. The configs have sane defaults and you can just drop the jar in without touching anything. If something breaks, hop in the Discord and I'll usually have a fix out the same day.

I'm not trying to compete with massive plugin teams. I just want to make a few focused plugins that are actually good, keep them updated, and not disappear after two months like half the stuff on SpigotMC.

Same /t and /land commands you already know from Towny and Lands. Configs have sane defaults. Drop the jar in, it just works. Bug reports in Discord usually get a fix same day.

Developer API

For building addons or integrating with other plugins.

PeaTowny API Addon System

Addons go in plugins/PeaTowny/addons/ and load automatically.

// grab the api
PeaTownyAPI api = PeaTownyAPI.getInstance();

// land queries
Land land = api.getLandAt(player.getLocation());
List<Land> lands = api.getLandsByOwner(playerId);

// permissions
boolean canBreak = api.hasPermission(landId, playerId, RoleFlag.BLOCK_BREAK);

// economy
api.deposit(landId, 500.0);
double balance = api.getBalance(landId);

// nations and wars
Nation nation = api.getNation(nationId);
List<War> wars = api.getActiveWars();

PeaSkills Commands

// player
/skills              - open skills menu

// admin
/skilladmin reload                           - reload config
/skilladmin setlevel <player> <skill> <level>  - set skill level
/skilladmin addxp <player> <skill> <amount>    - add xp
/skilladmin reset <player> [skill]            - reset skills
